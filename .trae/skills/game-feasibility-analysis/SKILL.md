# 游戏可行性分析

## 职责
负责分析游戏功能的技术可行性，评估实现难度、所需资源和时间成本。

## 工作流程

1. **需求理解**
   - 明确用户想要实现的游戏功能
   - 了解目标平台（PC、移动端、主机等）
   - 确认技术栈限制

2. **技术可行性评估**
   - 分析功能的技术实现路径
   - 评估现有技术栈是否支持
   - 识别潜在的技术难点

3. **资源评估**
   - 估算开发时间
   - 评估所需人力
   - 识别外部依赖（第三方库、插件等）

4. **风险评估**
   - 识别技术风险点
   - 评估性能影响
   - 考虑兼容性问题

5. **输出建议**
   - 给出可行性结论（可行/部分可行/不可行）
   - 提供替代方案（如需要）
   - 列出实现建议和注意事项

## 输出格式

```
## 可行性分析报告

### 功能概述
[简要描述要分析的功能]

### 技术可行性
- 可行性等级：⭐⭐⭐⭐⭐ (1-5星)
- 实现难度：低/中/高
- 技术栈要求：[所需技术]

### 实现方案
1. [方案1]
2. [方案2]

### 资源评估
- 预估开发时间：X天/周
- 所需技能：[技能列表]
- 外部依赖：[依赖列表]

### 风险评估
- 技术风险：[风险点]
- 性能影响：[影响评估]
- 兼容性问题：[兼容性问题]

### 建议
[综合建议和下一步行动]
```

## 注意事项
- 保持客观，不要过度承诺
- 考虑项目的实际约束条件
- 提供多个实现方案供选择
- 关注长期维护成本
