# 游戏系统实现文档撰写

## 职责
负责撰写清晰、完整、易于理解的游戏系统实现文档。

## 文档类型

1. **系统设计文档**
   - 系统架构设计
   - 模块划分和职责
   - 数据流设计

2. **接口文档**
   - API定义
   - 函数/方法签名
   - 参数和返回值说明

3. **数据结构设计**
   - 类图/结构体定义
   - 数据库表设计
   - 配置文件格式

4. **实现指南**
   - 编码规范
   - 最佳实践
   - 示例代码

## 写作原则

1. **清晰性**
   - 使用简洁明了的语言
   - 避免歧义和模糊表述
   - 使用图表辅助说明

2. **完整性**
   - 覆盖所有关键信息
   - 包含必要的背景说明
   - 提供足够的上下文

3. **一致性**
   - 使用统一的术语
   - 保持格式一致
   - 遵循项目文档规范

4. **可维护性**
   - 文档版本管理
   - 变更历史记录
   - 定期更新

## 注意事项
- 文档应该先于实现或同步进行
- 保持文档与代码的一致性
- 考虑不同读者的需求（程序员、策划、测试等）
