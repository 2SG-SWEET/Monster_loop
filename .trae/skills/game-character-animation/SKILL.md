# 角色帧动画资源设计

## 职责
负责设计和规划游戏角色的帧动画资源，包括动画规格、状态机和事件配置。

## 工作范围

1. **动画需求分析**
   - 角色类型和特点
   - 游戏玩法需求
   - 动画风格定义

2. **动画规格设计**
   - 动画列表定义
   - 帧率规范
   - 分辨率规格
   - 文件格式

3. **动画状态机设计**
   - 状态定义
   - 状态转换
   - 混合树配置

4. **动画事件配置**
   - 关键帧事件
   - 音效触发点
   - 特效触发点

## 动画类型

### 基础动画
- Idle（待机）
- Walk（行走）
- Run（奔跑）
- Jump（跳跃）
- Fall（下落）
- Land（落地）

### 战斗动画
- Attack（攻击）
- Hit（受击）
- Block（格挡）
- Dodge（闪避）
- Death（死亡）
- Skill（技能）

### 交互动画
- Interact（交互）
- Pickup（拾取）
- Use（使用）
- Emote（表情）

## 输出格式

### 动画规格文档

```markdown
# [角色名称] 动画规格

## 基本信息
- 角色类型：[类型]
- 风格：[风格描述]
- 参考分辨率：[分辨率]
- 标准帧率：[帧率]

## 动画列表

### 1. Idle（待机）
- 动画名称：Idle
- 持续时间：循环
- 帧数：8帧
- 帧率：12fps
- 描述：角色站立呼吸动画
- 循环：是
- 根运动：否
- 事件：无

### 2. Walk（行走）
- 动画名称：Walk
- 持续时间：循环
- 帧数：8帧
- 帧率：12fps
- 描述：角色行走循环
- 循环：是
- 根运动：是
- 事件：
  - 第2帧：Footstep_L
  - 第6帧：Footstep_R

### 3. Attack（攻击）
- 动画名称：Attack_01
- 持续时间：0.5秒
- 帧数：12帧
- 帧率：24fps
- 描述：普通攻击
- 循环：否
- 根运动：是
- 事件：
  - 第6帧：AttackStart（攻击判定开始）
  - 第8帧：AttackHit（攻击命中点）
  - 第10帧：AttackEnd（攻击判定结束）
  - 第6帧：PlaySFX（播放音效）

## 动画状态机

### 状态定义
- Grounded
  - Idle
  - Walk
  - Run
- Airborne
  - Jump
  - Fall
- Combat
  - Attack
  - Hit
  - Block

### 状态转换
- Any State → Death（触发条件：HP <= 0）
- Grounded → Jump（触发条件：Jump按钮）
- Jump → Fall（触发条件：速度向下）
- Fall → Land（触发条件：接触地面）
- Idle → Walk（触发条件：水平输入 != 0）
- Walk → Run（触发条件：按住Run + 水平输入 != 0）
- Any State → Attack（触发条件：Attack按钮）
```

## 命名规范

### 文件命名
```
[角色名]_[动画类型]_[变体编号]_[方向]

示例：
Player_Idle_01.png
Player_Walk_01_Left.png
Player_Attack_01.png
Enemy_Slime_Hit_01.png
```

### 动画剪辑命名
```
[动作]_[变体]_[方向]

示例：
Idle
Walk_Forward
Attack_01
Skill_Fire_01
```

## 技术规范

### Sprite Sheet 规范
- 使用统一的网格布局
- 保持一致的间距
- 提供透明度支持
- 考虑内存优化（图集打包）

### 动画事件规范
- 使用标准化的事件名称
- 在关键帧设置事件
- 避免在循环点设置事件

## 注意事项
- 考虑动画的流畅性和可读性
- 保持风格一致性
- 优化文件大小
- 考虑不同设备的性能
- 预留扩展空间
