# Monster Loop (生态环) —— 关卡场景功能文档（跑道+卡牌放置）

# 一、关卡场景核心定位

本关卡场景核心复刻《Loop Hero》循环玩法，以“大富翁式矩形跑道为核心载体”，跑道标线按大富翁棋盘形式设计，标线区域为卡牌（地块模块）唯一可放置区域，结合像素化中世纪+克苏鲁风格，承载“生态塑造、循环探索、战斗触发”三大核心玩法，同时适配地块模块化设计，支持后续新增地块模块的无缝接入，不改变场景核心逻辑。

核心目标：明确矩形跑道结构、大富翁式标线设计、卡牌放置规则、场景交互逻辑，确保程序可落地，同时贴合整体游戏风格，实现“跑道循环+标线放置+地块生效”的闭环。

# 二、跑道基础结构（程序可落地细节）

## 2.1 整体结构（适配Godot 4开发）

- 跑道形态：大富翁式矩形跑道（闭合矩形，四角为圆角过渡），采用Path2D节点绘制，路径闭环无断点，确保玩家可沿矩形跑道无限循环移动，贴合大富翁“绕棋盘循环”的核心形态；

- 尺寸参数（可配置）：
        

    - 矩形跑道整体尺寸：600×400像素（长×宽）；

    - 跑道宽度：80像素（标线放置区域+玩家移动区域，其中标线放置区域占40像素，玩家移动区域占40像素）；

    - 四角圆角半径：40像素（避免直角生硬，贴合大富翁棋盘圆润感）；

    - 放置格子划分：沿矩形跑道标线区域，均匀划分12个固定放置格子（大富翁棋盘式分布，每边3个格子，四角各1个格子），格子尺寸40×40像素，与标线宽度匹配；

    - 跑道像素精度：与整体风格统一，采用8-bit/16-bit像素，边缘做粗糙描边处理，贴合中世纪废墟质感。

- 节点层级（挂载在MapManager节点下）：
        

    - Path2D（命名：LoopTrackPath）：绘制矩形闭合跑道中心线（四角圆角过渡），绑定PathFollow2D（玩家移动控制器），控制玩家沿矩形跑道循环位移；

    - TileMapLayer（命名：TrackBackground）：渲染跑道背景，像素化石砖纹理，色调为深棕+暗灰，贴合中世纪风格，整体呈现大富翁棋盘底色；

    - Node2D（命名：TrackStrokeArea）：跑道标线放置区域，作为卡牌（地块模块）放置的唯一父节点，绑定碰撞检测区域，按大富翁格子分布划分碰撞范围；

    - Sprite2D（命名：TrackStroke）：跑道标线视觉表现（大富翁式），像素化粗描边（宽度4像素），颜色为暗金+暗红，按格子边界划分标线，每个格子对应一段独立标线，凸显放置区域，与背景区分；

    - Sprite2D（命名：TrackCornerMark）：矩形跑道四角标记，像素化中世纪风格符号（如小祭坛、扭曲符文），贴合大富翁棋盘“角落特殊格子”的设定，无实际功能，仅作视觉点缀；

    - ObjectPool（命名：TrackTilePool）：预加载所有地块模块预制体，用于标线区域的卡牌实例化，减少性能消耗。

# 三、核心功能：跑道标线区域卡牌放置（核心交互）

## 3.1 放置区域规则（程序判定逻辑）

- 放置范围：仅允许在“TrackStrokeArea”节点对应的跑道标线区域（12个大富翁式格子）放置卡牌（地块模块），超出标线区域的拖动、点击操作无效，无任何反馈；

- 放置格子划分：沿矩形跑道标线区域，按大富翁棋盘形式均匀划分12个固定放置格子（每边3个，四角各1个），每个格子对应1段标线、1个卡牌放置位，坐标固定，程序可直接读取；

- 格子限制：每个放置格子仅可放置1张卡牌（1个地块模块实例），不可叠加、不可替换，放置后卡牌（地块模块）与格子坐标对齐，贴合对应段标线，与大富翁“每格放1个道具/地块”逻辑一致；

- 放置前提：玩家拥有手牌（卡包配置的5张地块卡牌），支持两种放置方式——点击选中手牌后点击放置、直接拖动手牌至可放置区域，两种方式均需满足格子空闲条件方可触发。

## 3.2 放置交互流程（程序可直接调用）

支持两种放置方式（点击放置+拖动放置），核心逻辑一致，拖动放置新增拖动反馈与实时提示，具体流程如下：

### 方式一：点击放置（保留优化，适配矩形标线）

1. 选中手牌：玩家点击左侧侧边栏的地块卡牌（羊皮卷像素风格），卡牌触发高亮特效（像素描边+符文闪烁），视为选中状态，同时锁定该卡牌，不可重复选中；

2. 放置判定：玩家选中卡牌后，鼠标/触摸移动至跑道标线区域，若移动至空闲格子（对应段标线），该格子对应的标线立即亮起（像素化符文高亮，描边变亮）；若移动至已放置卡牌的格子或非标线区域，无任何提示，点击无效；

3. 实例化地块：玩家点击空闲格子，触发放置逻辑，消耗1张选中的手牌，从ObjectPool中取出对应地块模块预制体，在该格子实例化，模块位置与格子坐标完全对齐，贴合对应段标线；

4. 放置反馈：放置成功后，播放简单像素特效（符文爆发，1秒），同时手牌中对应卡牌消失，左侧侧边栏手牌自动更新；若手牌耗尽，放置功能禁用，标线区域不再显示“可放置”提示；

5. 异常处理：点击非标线区域、已占用格子、未选中手牌时点击标线区域，均无任何交互反馈，不消耗手牌、不实例化模块。

### 方式二：拖动放置（核心交互，适配矩形标线）

1. 拖动触发：玩家长按左侧侧边栏的地块卡牌（羊皮卷像素风格），触发拖动逻辑，卡牌跟随鼠标/触摸移动，同时卡牌自身高亮（像素描边+符文闪烁），侧边栏原卡牌位置显示临时占位符（半透明像素纹理）；

2. 实时提示：拖动卡牌过程中，鼠标/触摸移动至跑道标线区域时，程序实时判定下方格子状态——若为空闲格子，该格子对应的标线立即亮起（描边变亮+像素符文高亮，亮度高于点击放置的提示效果），明确提示可放置；若为已占用格子或非标线区域，无任何亮起提示，卡牌拖动时保持跟随，不触发任何判定；

3. 放置确认：拖动卡牌至亮起的空闲格子（对应段标线），松开鼠标/触摸，触发放置逻辑，消耗1张拖动的手牌，从ObjectPool中取出对应地块模块预制体，在该格子实例化，模块位置与格子坐标完全对齐；

4. 放置反馈：放置成功后，播放像素特效（符文爆发，1秒），侧边栏占位符消失，手牌列表自动更新；若拖动至非可放置区域松开，卡牌自动返回原手牌位置，占位符消失，不消耗手牌、不实例化模块；

5. 拖动取消：拖动过程中，若玩家按下取消键（或拖动至屏幕边缘），拖动逻辑终止，卡牌返回原手牌位置，占位符消失，不触发任何放置操作。

## 3.3 卡牌（地块模块）与跑道的联动逻辑

- 碰撞联动：玩家沿矩形跑道中心线（Path2D）循环移动，经过标线区域的地块模块时，触发模块的碰撞检测（CollisionShape2D），调用模块自身的consume_charge()方法，消耗Charge值（与原有逻辑一致）；矩形跑道四角格子的模块，玩家经过时无额外判定，与普通格子逻辑一致；

- 视觉联动：地块模块实例化后，贴合对应段标线显示，模块像素尺寸与格子尺寸匹配（40×40像素），不超出标线范围；模块特效（如微光森林的绿叶、熔岩裂隙的火焰）贴合标线，不遮挡玩家移动路径；

- 状态联动：地块模块Charge归0时，触发on_disappear()方法，模块销毁（播放对应像素特效），该格子恢复为空闲状态，标线对应段重新显示“可放置”提示（若玩家还有手牌）；

- 协同联动：标线区域相邻格子的地块模块，触发协同效应时（如水系+电系），相邻格子对应的标线同步闪烁（对应特效颜色），提示玩家当前存在模块协同效果，贴合大富翁“相邻格子联动”的视觉逻辑。

# 四、跑道循环与场景交互补充

## 4.1 玩家循环移动（与矩形跑道适配）

- 位移控制：PathFollow2D绑定玩家角色，沿矩形跑道中心线（Path2D）循环移动，速度固定为150像素/秒（可配置），移动方向为顺时针（固定，不可修改），四角区域自动平滑转弯，无卡顿；

- 循环判定：当PathFollow2D的progress达到1.0（矩形跑道终点，与起点重合，位于矩形左下角）时，自动重置为0.0，完成1圈循环，同时触发相关逻辑（如孵化器进度+1、熔岩裂隙模块的ATK加成）；

- 移动限制：玩家仅可沿矩形跑道中心线移动，不可偏离跑道（程序限制PathFollow2D的移动范围，超出范围自动拉回中心线），确保碰撞地块模块的准确性，尤其注意四角区域的移动贴合；

- 暂停机制：触发战斗、放置卡牌时，玩家移动暂停；战斗结束、放置完成后，自动恢复移动，不打断循环节奏。

## 4.2 场景辅助交互

- 放置提示：优化提示逻辑，适配拖动放置与矩形标线——玩家拖动卡牌时，空闲格子对应的标线实时亮起（描边+符文高亮），已占用格子无提示；点击选中手牌时，空闲格子对应的标线持续显示“可放置”提示（符文高亮），已占用格子显示模块图标缩略图（像素化）；两种放置方式的提示效果统一风格，拖动放置的亮起提示亮度更高，区分交互状态；

- 模块信息查看：点击标线区域已放置的地块模块，弹出简易信息面板（像素风格），显示模块名称、当前Charge值、核心特效（如“偷窃成功率+20%”），点击面板外区域关闭；

- BOSS进度联动：跑道顶部显示BOSS觉醒进度条（像素化，暗金描边），进度同步地块模块Charge消耗情况，进度达到100%时，跑道起点（Path2D起点，矩形左下角）生成BOSS预制体，同时整个跑道标线区域闪烁红光（警示特效），贴合大富翁“终点触发特殊事件”的逻辑；

- 风格氛围交互：玩家每完成1圈循环，跑道周围的迷雾特效轻微变化（浓度波动），标线符文闪烁频率加快，强化克苏鲁未知氛围；矩形四角的标记符号，在玩家经过时会轻微闪烁，增加大富翁棋盘的互动感。

# 五、模块化适配与拓展性

- 地块模块适配：所有地块模块预制体尺寸统一为40×40像素，适配矩形标线的12个放置格子，模块脚本无需修改，仅需确保碰撞范围与模块尺寸一致，即可无缝接入跑道标线放置系统；

- 跑道拓展：后续可修改Path2D节点的路径形态（如调整矩形尺寸、增加格子数量），标线区域会自动适配新路径，放置格子数量可通过程序配置调整，无需修改核心放置逻辑；大富翁式标线可根据新增格子数量，自动划分对应段数；

- 风格拓展：新增地块模块时，仅需适配像素中世纪+克苏鲁风格，模块视觉表现与跑道背景、标线效果统一即可，不影响放置、碰撞等核心功能；

- 性能优化：标线区域的放置格子坐标固定，地块模块实例化采用对象池管理，避免频繁创建/销毁节点；玩家移动与碰撞检测采用Path2D自带的坐标判定，降低性能消耗；矩形跑道的碰撞检测的范围更规整，可进一步优化检测效率。

# 六、开发优先级与测试要点

## 6.1 开发优先级（贴合Demo整体进度）

1. 优先级1（MVP 1阶段）：完成矩形跑道基础结构（Path2D、背景、大富翁式标线），实现玩家沿矩形跑道循环移动，不添加卡牌放置功能；重点调试四角区域的平滑转弯；

2. 优先级2（MVP 2阶段）：完成标线区域12个放置格子划分、手牌选中与放置逻辑（点击+拖动），实现地块模块在标线区域的实例化、碰撞检测与Charge消耗；

3. 优先级3（MVP 3阶段）：完善放置提示、模块信息查看、风格特效（迷雾、符文闪烁、四角标记闪烁），适配BOSS进度联动与模块协同效应的视觉反馈。

## 6.2 测试要点

- 放置逻辑测试：确保拖动放置与点击放置均生效，拖动时可放置区域（对应标线）精准亮起，非可放置区域无提示；拖动取消、放置失败时卡牌正常返回，不出现卡顿或误放置；
